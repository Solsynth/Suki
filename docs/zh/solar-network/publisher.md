---
title: 发布者和发布内容
description: 了解在 Solar Network 上发表内容。
---

发布者是 Solar Network 上大多数公开发表的内容的管理单位，
一个帐号可以拥有 / 加入多个发布者。

在 Solar Network 发布任何内容之前，你需要一个发布者身份。

## 创建

你可以在发布帖子页面点击空白的（在未创建发布者时可用）发布者头像来创建、
或者通过「帐号」页面前往「创作者中心」创建（桌面端可直接在侧边栏访问「创作者中心」标签）

!!! note

      若在创建发布者时出现 `Permission required` 的提示即表示你缺少部份权限。
      通常来说，用户需要接受一个确认注册的邮件来确认注册，若未确认注册的帐号不会拥有创建发布者的权限。

一个用户可以同时拥有多个发布者，也可以加入不同的发布者共同管理。

在创建发布者时，你需要注意以下几点：

1. 名称 —— 与帐号名称一样，需要全站唯一并且保持 URL-safe，创建后不可更改。
2. 头像与头图 —— 在创建、编辑发布者的时候，头像与头图的上传并不是上传之后就立刻应用，
   需要点击 “保存更改” 来正式应用。

## 个人与组织

发布者分为两种类型，一个是「个人」，一个是「组织」。个人和组织发布者都可以邀请他人共同管理。
其最大的不同是「个人」发布者会在发布者信息缺失的时候（如头像、头图等）使用用户的信息作为回退版本，
而领域则不会。同时「个人」发布者还会显示用户的状态、和认证状态。

同时「组织」发布者的头像是特殊的圆角矩形而并非圆形。
并且「组织」发布者需要隶属于一个「领域」。

## 合作

自 v3 以来，发布者可以邀请他人共同管理，内设四个默认的权限等级：

1. 所有者 —— 拥有所有权限
2. 管理者 —— 拥有编辑者、观察者的所有权限，同时可以邀请新用户共同管理（权限不能高于自身）
3. 编辑者 —— 拥有在这个发布者上发布资源的权限
4. 观察者 —— 拥有查看这个发布者的内部统计信息的权限

## 开发者

「开发者」是「发布者」的一种特殊变种，其可以在「开发者门户」创建「发布者」的「开发者身份」，
用来调用 Solar Network 的开放资源。

详情请前往「开发者身份」章节查看。

## 认证标志

如果你有在 Solar Network 被他人冒充的疑虑，或是想发布一些权威信息，亦或是想展示自己的独特性。你可以申请认证标志。

> 你需要在申请认证时拥有 “恒星计划” 订阅。否则请求将不会被处理。认证标志在 “恒星计划” 订阅过期后不会消失。

申请发布者的认证与申请帐号的验证相同，你需要向 `lily[at]solsynth.dev` 发送邮件说明，客户支持团队会在 7 个工作日之内回复处理。

> 您若需要申请官方性质的认证，需要提供一些证明材料。越齐全成功率越高。

## 可见性

可见性设置适用于帖子，不过不同类型的发布者会有不同的可见性可用性。

- 公开：所有人都可以看见，包括未登陆用户和 API
- 仅好友可见：只在个人发布者上生效，会让帖子只给发布者创建者的好友可见。当在组织发布者上使用时，其会等同于私人。
- 私人：只有发布者自己可见。
- 未列出：帖子不会出现在推荐流和列表中（发布者自己除外）但是用户仍然可以通过帖子 ID 或者别名直接访问。

### 领域帖子

领域帖子在首页上的显示与否会根据用户加入领域与否，以及领域的设置决定。

- 若领域启用了公开模式，则领域帖子会给未加入领域的用户显示。
- 若领域没有启用公开模式，则领域帖子只会在已加入用户的主页推荐流显示。

!!! warning

      领域帖子仍然尊重原本的可见性设置。并且领域的设置只会影响推荐（以及一般的不排序列出帖子 API）与否，
      不会影响帖子本身的可见性。当用户带有领域的查询参数对领域帖子进行查询，仍然可以得到完整信息。
      包括领域帖子也会显示在发布者页面。

### 有门槛的发布者

「关注需批准」和「发布内容需关注后可见」是恒星计划 Nova 或以上等级的特别功能。允许你进行更细致的设置关于部份用户可见与否。

对于「发布内容需关注后可见」的发布者，用户不会直接在主页看到这名发布者的内容，除非用户是关注用户。

对于「关注需批准」的发布者，用户不会在发布者页面直接关注用户，而是进入一个等待（Pending）阶段。
直到发布者审核之后才会成为真正的订阅者。若用户之后主动取消关注，则需要重新申请关注。

额外的，对于「关注需批准」的发布者，用户可以手动添加用户到关注列表。但是注意：

- 如果用户手动取消关注过，则不可以再次添加
- 手动添加的关注不会默认启用通知。

!!! warning

      帖子自身的可见性设定会覆盖「关注需批准」。若你的帖子标记了仅好友可见，未关注的好友仍然可见。

## 订阅发布者

你可以订阅发布者，以查看「发布内容需关注后可见」的发布者内容。

自从「发布内容需关注后可见」功能上线之后，我们为订阅添加了通知等级的选项，
若你只想保留可见权限。可以将通知选项关闭。

## 联邦角色

Solar Network 的联邦化是根据发布者来运作的。所以你至少要有一个联邦化的发布者才能对联邦帖子/内容进行互动。

联邦化的内容中只会涉及你的公开内容。不过「关注需批准」的内容仍然会发送给联邦服务器，
因此若你不希望这点，请关闭对应发布者的联邦功能。

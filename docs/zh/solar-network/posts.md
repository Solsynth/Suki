帖子是在探索页面会出现的主要内容。帖子隶属于发布者，会尊重发布者的设置。

## 内容

Solar Network 的帖子支持全部的 CommonMark 和部份的 GitHub Flavoured Markdown 以及一些自定义的语法。

### Solar Network Flavoured Markdown

本章节会向你介绍 Solar Network 特别的 Markdown 语法

#### 高亮

和 obsidian 类似，使用 `==Text==` 语法

#### 隐藏/剧透

使用 `=!Text!=` 语法。单击以显示。

!!! warning

    在帖子中使用这个语法请在开头先写一些文字（80 字左右）不然可能通知会让你的剧透内容"被剧透"。

#### 贴图

你在帖子中也照样能使用贴图，直接填写占位符即可，例如 `:prefix+slug:`

#### 帖子内附件

和 Markdown 的语法一样使用 `![](url)`

但是 URL 处填写 `solian://files/<id>` 会加载我们自有的附件预览，获得更好的显示效果。包括支持放大等

### 附件

Solar Network 帖子的附件支持的文件类型包罗万象，从图片到语音再到文件。
全都支持，并且支持混种使用，一次性至多添加 16 个！<del>遥遥领先。</del>

对于只有图片的帖子，客户端会采用特色的轮播图来显示；对于混种附件的帖子，客户端会回退到一般横向滚动列表来保证可用性。

### 标签与分类

给你的帖子添加标签与分类可以帮助我们的推荐算法讲你的帖子推荐给感兴趣的人，增加你的初级面积。

但是滥用帖子和标签会违反 ToS，造成收到警告或者停权，不要淘气哦。

## 动态排名算法

Solar Network 的动态（时间线）排名采用多层次打分系统，确保为您推荐最相关、最有趣的内容。

### 排名模式

您可以选择不同的浏览模式：

| 模式 | 说明 |
|------|------|
| 个性化 | 根据您的兴趣推荐内容（默认） |
| 热门 | 按互动热度排序 |
| 最新 | 按发布时间倒序排列 |

### 评分因素

#### 基础得分

帖子首先会根据以下因素计算基础得分：

- **反应得分**：根据反应的态度计算
  - 正面反应：+2 分
  - 中立反应：+1 分
  - 负面反应：-2 分
- **回复数**：帖子收到的回复数量
- **赞赏得分**：帖子收到的赞赏
- **文章加成**：文章类帖子会获得额外的基础分
- **时间衰减**：随着时间推移，旧帖子的得分会逐渐降低

#### 个性化得分

对于已登录的用户，系统会根据您的兴趣进一步调整排名：

**兴趣来源**：

- 您对帖子添加的反应
- 您发表的回复
- 您查看过的帖子（每天每用户有限制）
- 您明确标记的「感兴趣」或「不感兴趣」

**兴趣标签**：
系统会追踪您对以下内容的兴趣：

- 标签
- 分类
- 发布者

兴趣会随时间衰减，所以您近期的行为比早期行为更能影响推荐。

#### 发布者加成

发布者的社会信用等级也会为帖子带来额外的排名加成。信用等级越高的发布者，其帖子在推荐中会获得小幅提升。

#### 筛选与多样化

在个性化模式下，系统还会进行：

1. **强度过滤**：过滤掉与您兴趣匹配度较低的帖子
2. **多样性保护**：确保动态中不会只有同一个发布者的内容

### 反馈与调整

您可以通过以下方式影响推荐：

- 对帖子点「赞」或「踩」
- 对发布者选择「隐藏」
- 在设置中调整兴趣偏好

这些操作会直接影响您后续看到的推荐内容。

### 为什么我的帖子没被推荐？

如果您的帖子没有被推荐，可能有以下原因：

1. **发布时间较晚**：时间衰减会导致旧帖子得分降低
2. **互动较少**：反应、回复、赞赏都会影响得分
3. **兴趣匹配度低**：您的帖子可能与目标受众的兴趣不匹配
4. **发布者信用分较低**：这会影响基础排名加成
5. **被标记为不感兴趣**：如果您或您的关注者曾对类似内容表示不感兴趣

## 发布设置

您可以配置发布帖子、回复和联邦网络活动时使用的默认发布者。

### 设置位置

进入 **账号** > **账号设置** > **发布设置**

### 可配置的项目

| 设置 | 说明 |
|------|------|
| 默认发帖发布者 | 发布新帖子时使用的发布者 |
| 默认回复发布者 | 回复帖子时使用的发布者 |
| 默认联邦网络发布者 | 在联邦网络（如 Mastodon）进行关注、转推、反应时使用的发布者 |

### 设置优先级

当您执行操作时，系统会按以下顺序选择发布者：

1. **手动选择**：如果您在发帖时明确选择了发布者，以您的选择为准
2. **默认设置**：如果您配置了默认发布者，使用您设置的默认值
3. **自动选择**：使用您拥有的第一个符合条件的发布者

### 使用场景示例

**场景 1：个人与组织身份分开**

您有两个发布者：

- `@我的个人账号` - 个人发布者
- `@我的公司` - 组织发布者

您希望：

- 发布个人内容时使用 `@我的个人账号`
- 回复他人时使用 `@我的公司`
- 在联邦网络互动时使用 `@我的公司`

只需在发布设置中分别配置这三项即可。

**场景 2：只有一个发布者**

如果您只有一个发布者，系统会自动使用它，无需额外配置。

**场景 3：联邦网络发布者**

配置默认联邦网络发布者后，您在 Mastodon 等平台进行以下操作时会自动使用该发布者：

- 关注其他用户
- 取消关注
- 转推帖子
- 对帖子添加反应

### 注意事项

- 您必须是发布者的成员才能将其设为默认值
- 联邦网络发布者必须先启用联邦网络功能才能设置为默认值
- 如果您设置的联邦网络发布者后来禁用了该功能，系统会自动回退到其他可用的发布者
